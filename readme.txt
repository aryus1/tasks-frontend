--Documentação de como usar a arquitetura de pastas--

public: arquivos publicos (favicon, index.html, etc)

src/assets: arquivos estáticos (imagens, logos, fontes)

src/components: componentes para serem reaproveitados (botões, formularios, cabeçalhos, etc)

src/context: context API, para gereciamento global

src/hooks: hooks reacts personalizados

src/pages: páginas web do projeto (Tela de login, tela de dashboard, etc)

src/services: funções para consumir do backend

src/styles: estilos css

src/utils: funções utilitárias (formatação de datas, etc)
